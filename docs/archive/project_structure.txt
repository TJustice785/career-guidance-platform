## File Creation Order

### Priority 1: Core Setup Files (Start Here)
1. `README.md`
2. `.gitignore`
3. `server/package.json`
4. `client/package.json`
5. `server/.env` (from template)
6. `client/.env` (from template)

### Priority 2: Backend Foundation
7. `server/src/config/firebase.config.js`
8. `server/src/index.js`
9. `server/src/middleware/auth.middleware.js`
10. `server/src/services/email.service.js`

### Priority 3: Authentication
11. `server/src/routes/auth.routes.js`
12. `server/src/controllers/auth.controller.js`

### Priority 4: Module Routes & Controllers
13. `server/src/routes/student.routes.js`
14. `server/src/controllers/student.controller.js`
15. `server/src/routes/institute.routes.js`
16. `server/src/controllers/institute.controller.js`
17. `server/src/routes/company.routes.js`
18. `server/src/controllers/company.controller.js`
19. `server/src/routes/admin.routes.js`
20. `server/src/controllers/admin.controller.js`

### Priority 5: Frontend Setup
21. `client/tailwind.config.js`
22. `client/src/index.css`
23. `client/src/config/firebase.config.js`
24. `client/src/services/api.service.js`

### Priority 6: Main App & Pages
26. `client/src/App.js`
27. `client/src/index.js`
28. `client/src/pages/LandingPage.js`
29. `client/src/pages/LoginPage.js`

### Priority 7: Dashboard Pages
31. `client/src/pages/student/StudentDashboard.js`

33. `client/src/pages/company/CompanyDashboard.js`

### Priority 8: Components (As Needed)
35. Common components (Button, Card, Modal, etc.)
36. Role-specific components
37. Feature-specific pages

## Quick Start Commands

### Windows (PowerShell)
```powershell
# Create main directories
New-Item -ItemType Directory -Path server\src\config,server\src\controllers,server\src\routes,server\src\middleware,server\src\services,server\src\utils,server\src\models,server\src\scripts
New-Item -ItemType Directory -Path client\src\components\common,client\src\components\admin,client\src\components\student,client\src\components\institute,client\src\components\company
New-Item -ItemType Directory -Path client\src\pages\admin,client\src\pages\student,client\src\pages\institute,client\src\pages\company
New-Item -ItemType Directory -Path client\src\contexts,client\src\services,client\src\utils,client\src\hooks,client\src\config

# Install dependencies
cd server
npm install
cd ../client
npm install
cd ..
```

### Unix/Mac/Linux
```bash
# Make setup script executable
chmod +x setup.sh

# Run setup script
./setup.sh
```

## Development Workflow

1. **Open VS Code**
   ```bash
   code .
   ```

2. **Open Two Terminals in VS Code**
   - Terminal 1 (Backend):
     ```bash
     cd server
     npm run dev
     ```
   - Terminal 2 (Frontend):
     ```bash
     cd client
     npm start
     ```

3. **Access Application**
   - Frontend: http://localhost:3000
   - Backend API: http://localhost:5000/api

## Notes

- All `.env` files must be created manually and never committed to Git
- Use the provided templates to create your environment files
- Follow the Priority order for file creation to avoid errors
- Test each module after implementation before moving to the next
